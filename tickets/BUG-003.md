## Description

The loan feature (lending money to a friend, tracking outstanding, receiving it back) isn't working as designed. Issues:

1. When creating a transfer to/from an external account (person), it should automatically create a transfer pair relation and mark it as outstanding. Currently the user would have to manually create a relation and add members — that's not acceptable UX.

2. Lending money should NOT require a category selection, or should have a fixed category like "loan" or "transfer". Currently it uses the same category dropdown as expenses.

3. The Outstanding Transfers page doesn't have functional actions — user can't easily "log the return" from there.

## Expected behavior

1. When user creates a transfer where from_account OR to_account is an external (person) account:
   - Automatically create a transfer_pair relation
   - Add this transaction as the "outgoing" member
   - It shows up in Outstanding Transfers immediately

2. Transfer transactions to/from people: category should auto-set to "transfer" or "loan" (not user-selectable from the regular category list)

3. Outstanding Transfers page: "Log return" button should open the transaction form pre-filled with:
   - Type: transfer
   - Amount: same as original
   - From/to: swapped from original
   - After saving, automatically links as "incoming" member to the same pair

## Acceptance

- Create transfer to "Max" → automatically appears in Outstanding
- Click "Log return" on outstanding item → pre-filled form → save → pair is complete → disappears from Outstanding
- Category is auto-assigned for person-to-person transfers
- Full E2E verified
