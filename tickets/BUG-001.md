## Description

After the API contract mismatch fix (PR fix/api-contract-mismatch), the environment variable `VITE_API_URL` no longer works correctly. The frontend may be ignoring it or it's hardcoded somewhere.

## Steps to reproduce

1. Set VITE_API_URL to a non-default value
2. Build or run the frontend
3. Observe API calls still go to hardcoded URL

## Expected

Frontend respects VITE_API_URL for all API calls.

## Acceptance

- VITE_API_URL is used consistently across all API modules
- Docker Compose setup works with the env var
- Verified by running with a custom VITE_API_URL and confirming requests go to the right host
