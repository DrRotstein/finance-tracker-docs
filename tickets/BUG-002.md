## Description

Creating transaction groups doesn't work. Additionally, the backend requires a `label` field for groups, but per user requirements, groups should not require a name (label should be optional).

## Steps to reproduce

1. Try to create a group via POST /relations/group without a label
2. Observe it fails (400 or similar)

## Expected

- Groups can be created without a label (label is optional)
- Groups can be created with a label
- Both paths work end-to-end (API + UI)

## Acceptance

- POST /relations/group works with `{}` body (no label)
- POST /relations/group works with `{"label": "Dorm supplies"}` body
- Frontend group creation flow works
- Verified by E2E test
